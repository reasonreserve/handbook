# Handbook / service-template — GitHub tracking

## Source of truth

**GitHub is canonical:** org **Projects** (boards, fields, status) and **Issues** on [github.com/reasonreserve](https://github.com/reasonreserve) lead. This file is a **maintained mirror**—after you change titles, states, or project membership on GitHub, update this table so contributors are not misled.

- **Org projects:** [Projects for @reasonreserve](https://github.com/orgs/reasonreserve/projects) — programme-level work is on **[Portfolio](https://github.com/orgs/reasonreserve/projects/1)**; add handbook issues there when you want them on the stakeholder board.
- **Handbook backlog:** [reasonreserve/handbook issues](https://github.com/reasonreserve/handbook/issues) (below).

### Status snapshot

Refresh this section when the backlog or release cadence changes; **GitHub** stays authoritative.
